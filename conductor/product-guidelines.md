# Product Guidelines: Lumina Reader

## Tone & Prose Style
- **Friendly & Accessible**: Use a warm, encouraging, and user-centric tone in both documentation and user-facing copy. Instructions should be clear but welcoming.

## Visual Language & Branding
- **Glassmorphism & Vibrancy**: Incorporate frosted glass effects, soft shadows, and vibrant accent colors to create a modern, "premium" feel.
- **Skeuomorphic & Organic**: Use tactile textures, warm paper tones, and high-quality serif fonts to evoke the feeling of reading a physical book.
- **The Lumina Look**: The interface should feel "alive" through the use of subtle gradients, layering, and dynamic lighting effects.

## Core UX Principles
- **Content First**: The reading material is the star. Interface elements should be unobtrusive and hide automatically when not in use to provide a distraction-free environment.
- **Fluid Responsiveness**: Every user action should trigger instant visual feedback and a smooth transition. Animations should be purposeful and delightful, never distracting.
- **Consistency**: Maintain a unified design system across the library, settings, and reader views.

## Development Principles
- **Test-Driven Quality**: Adhere to a strict Test-Driven Development (TDD) workflow. Every task should include comprehensive unit and/or end-to-end (E2E) tests to ensure long-term stability.
- **Performance Profiling**: Regularly profile the application to ensure that animations remain smooth (60fps) and that the database handles large libraries efficiently.
- **Accessibility as a Standard**: Build accessibility features into the core development process, not as an afterthought. This includes keyboard navigation, ARIA labels, and support for specialized fonts.
